Introduction

HTML is a markup language that sets the content and structure of a website. All text in an HTML file should generally be wrapped in tags. What the tags represent are called elements. Elements serve a specific semantic purpose, and the content should align with this purpose.

Here is an example of a paragraph and an image tag:

<p>[Paragraph goes here, without brackets]</p> <img src="[Image link goes here, without brackets]" />
Elements

Elements serve a specific purpose. For example, these are header elements you would use to emphasize a main idea about the content that follows the header:

<h1>[This is a main header! Without brackets]</h1> <h2>[This is header 2! Without brackets]</h2>

Here is a list of some other elements:

  • <a>[This is a link tag]</a>
  • <div>[This is a div tag, which is used for styling purposes]</div>
  • <title>[The title of the webpage, which is nested inside of the head element. Not visible on the webpage itself.]</title>
  • <body>[An important tag included on every webpage. The content inside is what is displayed on the webpage.]</body>
  • <section>[A section tag that separates main parts of the webpage.</section>

Browsers will usually give certain elements default style properties, like h1 being displayed with a bold and big font. However, these styles should generally be ignored and removed (in css code) as HTML's main purpose is not to style but to semantically organize content.

Attributes

Attributes are additions to elements that can change their behavior or function. They are applied within the first tag of an element and have a value in quotes. Sometimes, they are a necessity for the element, and other times they will simply add extra text to the element.

Below is a link tag. The link tags do nothing on their own, but the href attribute directs the text to that link once interacted with.

<a href="https://google.com"></a>

You can add multiple attributes within one element, separated by spaces. "id", "src", and "alt" are all attributes added to the image element.

<img id="cat-photos" src="cats.png" alt="Image of cats" />
Self-Closing Tags

Typically, elements will have a beginning tag and an ending tag. However, self-closing tags are those that do not have an ending tag. Some examples include the meta or img tag.

Most tags: <h1>[Text here]</h1> <p>[This is a paragraph tag with an opening and closing p.]</p> Self-closing tags: <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <img src="cats.png" />
Semantics

When choosing an element for specific content, it is important to think about the importance of the content in relation to the element's role. As stated earlier, HTML code does not serve to style any content. Therefore, many different elements have the same innate properties but different semantic roles. Content should be assigned tags that best fit their purpose on the webpage.

A paragraph tag may be used for large blocks of text.

<p>This is a large paragraph that will serve to be a large paragraph. Why this paragraph is large is because paragraph tags are for blocks of text. Although the paragraph tag can be applied for small bits of text, it would not fit the semantic purpose of a paragraph element. It is important to follow common semantic practices to keep code organized and readable for humans.</p>
